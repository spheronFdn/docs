# Spheron Site SDK

The Spheron Site SDK is a npm package, that provides support for working with Spheron Web-App organizations.

<Callout type="info">
  **NOTE:** The package is only meant for Node.js environments.
</Callout>

## Installation

**Using NPM**

```
npm i @spheron/site
```

**Using Yarn**

```
yarn add @spheron/site
```

## Usage

To use the Spheron Site SDK, you have to create an instance of `SpheronClient`.

```js
import { SpheronClient, ProtocolEnum } from "@spheron/site";

const client = new SpheronClient({ token });
```

The `SpheronClient` constructor takes an object that has one property `token`.
Follow the instructions in the [DOCS](https://docs.spheron.network/rest-api/#creating-an-access-token) to generate this token.

<Callout type="info">
  **NOTE:** When you are creating the tokens, please choose **web app** type in
  the dashboard.
</Callout>

The `SpheronClient` has three properties:

- `organization` provides methods for working with organizations.
- `projects` provides methods for working with projects.
- `deployments` provides methods for working with deployments.

## Organization

The methods for working with organizations are available on the `organization` property of the `SpheronClient` object. All the methods work with the organization of the token that is passed in the constructor.

### get

Used to get the organization.

```js
const organization: Organization = await spheron.organization.get();
```

Returns an Organization for which the token was created for.

```js
interface Organization {
  id: string;
  profile: {
    name: string,
    username: string,
    image: string,
  };
}
```

---

### getProjects

Used to get the projects of the organization. The methods support pagination.

```js
const projects: Project[] = await spheron.organization.getProjects({
  skip: 0,
  limit: 10,
  state: ProjectStateEnum.MAINTAINED,
});
```

Params:

- `options`:
  - `skip`: number of projects to skip.
  - `limit`: number of projects to take.
  - `state` ( optional ): project state filter.

Returns an array of projects. Checkout the [get project section for more details on the project interface properties](/sdk/site/#get-1).

---

### getProjectCount

Used to get the number of projects in the organization.

```js
const count: number = await spheron.organization.getProjectCount({
  state: ProjectStateEnum.MAINTAINED,
});
```

Params:

- `options` ( optional ):
  - `state` ( optional ): project state filter.

Returns the number of projects.

---

### getUsage

Used to get the current subscription usage of the organization.

```js
const usage: UsageWithLimits = await spheron.organization.getUsage();
```

Returns the `UsageWithLimits` object.

```js
interface UsageWithLimits {
  usedBandwidth?: number; // the bytes of bandwidth used for the current subscription
  bandwidthLimit?: number; // the bandwidth limit for the current subscription
  usedBuildExecution?: number; // the seconds of build execution done for the current subscription
  buildExecutionLimit?: number; // the build execution limit for the current subscription
  usedConcurrentBuild?: number; // the number of concurrent builds in progress
  concurrentBuildLimit?: number; // how many concurrent builds can the organization have with the current subscription plan
  usedStorageArweave?: number; // the bytes of used Arweave storage for the current subscription
  storageArweaveLimit?: number; // the limit in bytes for the Arweave storage for the current subscription
  usedStorageIPFS?: number; // the bytes of used IPFS storage for the current subscription
  storageIPFSLimit?: number; // the limit in bytes for the IPFS storage for the current subscription
  usedDeploymentsPerDay?: number; // number of deployments an organization can have one one day
  deploymentsPerDayLimit?: number; // the limit of how many deployments can an organization have per day
  usedDomains?: number; // the number of domains and subdomain an organization has
  domainsLimit?: number; // the limit on how many domains and subdomains can an organization have
  usedHnsDomains?: number; // the number of hns domains an organization has
  hnsDomainsLimit?: number; //  the limit on how many hns domains can an organization have
  usedEnsDomains?: number; // the number of ens domains an organization has
  ensDomainsLimit?: number; // the limit on how many ens domains can an organization have
  usedEnvironments?: number; // the number of deployment environments an organization has
  environmentsLimit?: number; // the limit on how many deployment environment can an organization have
  usedNumberOfRequests?: number; // the number of requests the organization has had till now
  membersLimit?: number; // how many members can an organization have
  usedPasswordProtections?: number; // 1 - if organization has support for password protection, otherwise 0.
}
```

## Projects

### get

Used to get the project based on the passed Id.

```js
const project: Project = await spheron.projects.get(projectId);
```

Params:

- Id of the project.

Returns an object of type `Project`.

```js
interface Project {
  id: string;
  name: string;
  state: ProjectStateEnum;
  url: string; // the url to the git repository
  organizationId: string;
  provider: ProviderEnum;
  configuration: Configuration;
}

interface Configuration {
  installCommand: string; // command that will be used to install the dependencies of the repository
  buildCommand: string; // command that will be executed to build the repository
  workspace: string; // the path to the app inside repository
  publishDir: string; // the path to the build directory
  framework: FrameworkEnum;
  nodeVersion: NodeVersionEnum;
}

enum ProjectStateEnum {
    MAINTAINED = "MAINTAINED",
    ARCHIVED = "ARCHIVED"
}

enum ProviderEnum {
    GITHUB = "GITHUB",
    GITLAB = "GITLAB",
    BITBUCKET = "GITLAB"
}

enum NodeVersionEnum {
    V_12 = "V_12",
    V_14 = "V_14",
    V_16 = "V_16"
}

enum FrameworkEnum {
    SIMPLE_JAVASCRIPT_APP = "static",
    VUE = "vue",
    REACT = "react",
    NEXT = "next",
    ANGULAR = "angular",
    PREACT = "preact",
    NUXT = "nuxt2",
    SVELTE = "svelte",
    GATSBY = "gatsby",
    ELEVENTY = "eleventy",
    DOCUSAURUS = "docusaurus",
    SANITY = "sanity",
    HUGO = "hugo",
    IONIC_REACT = "ionicreact",
    VITE = "vite",
    SCULLY = "scully",
    STENCIL = "stencil",
    BRUNCH = "brunch",
    IONIC_ANGULAR = "ionicangular"
}
```

### archive

Used to archive a project. When a project is archived new deployments cannot be started for it. If a project is already in Archived state, an error will be thrown.

```js
await spheron.projects.archive(projectId);
```

Method return type is void. If there is an issue while archiving, an error will be thrown.

---

### unarchive

Used to update the project state from Archived to Maintained. If a project is already in Maintained state, an error will be thrown.

```js
await spheron.projects.unarchive(projectId);
```

Method return type is void. If there is an issue while archiving, an error will be thrown.

---

### updateConfiguration

Used to update the configuration of a project.

```js
const configuration: Configuration = await spheron.projects.updateConfiguration(
  projectId,
  {
    framework: FrameworkEnum.REACT,
    workspace: "",
    installCommand: "yarn install",
    buildCommand: "yarn build",
    publishDir: "build",
    nodeVersion: NodeVersionEnum.V_16,
  }
);
```

Params:

- Id of the project.
- `configuration`:

  - `framework`: the framework of the repository
  - `workspace`: the path to the app inside repository
  - `installCommand`: command that will be used to install the dependencies of the repository
  - `buildCommand`: command that will be executed to build the repository
  - `publishDir`: the path to the build directory
  - `nodeVersion`: the node version of your repository

Returns the updated configuration.

```js
interface Configuration {
  installCommand: string;
  buildCommand: string;
  workspace: string;
  publishDir: string;
  framework: FrameworkEnum;
  nodeVersion: NodeVersionEnum;
}
```

### addEnvironmentVariable

Used to add environment variables to a project.

```js
const added: EnvironmentVariable[] =
  await spheron.projects.addEnvironmentVariable(projectId, [
    {
      name: "variable1",
      value: "value1",
      environments: ["Production"],
    },
    {
      name: "variable2",
      value: "value2",
      environments: ["Development", "Production"],
    },
  ]);
```

Params:

- Id of the project.
- Array of variables that should be added. Each object of the array should have the following properties:
  - `name`: the name of the variable.
  - `value`: the value of the variable.
  - `environments`: the deployment environments for which the variable will be used

Returns an array of added variables.

### addDomain

Used to add a new domain to a project.

```js
const domain: Domain = await spheron.projects.addDomain(projectId, {
  name: "domain.com",
  type: DomainTypeEnum.DOMAIN,
  deploymentEnvironments: ["Production"],
});
```

Params:

- Id of the project.
- `options`:
  - `name`: the domain name.
  - `type`: the domain type.
  - `deploymentEnvironments` ( optional ): name of the deployment environments. Whenever a deployment for the specified deployment environment executed, the domain will be updated to point to the newly deployed data.
  - `link` ( optional ): The `sitePreview` of a specific deployment of the same project. When this property is set, the domain will point to data of the specified deployment, and will not be updated when a new deployment is done.

<Callout type="info">
  **NOTE:** Ether `deploymentEnvironments` or `link` is required in the request
  body.
</Callout>

Returns the newly created domain. After a domain is created you need to call the [verifyDomain](/sdk/site/#verifydomain) function for it to work.

### deploy

Used to start a new deployment.

```js
await spheron.deployments.deploy({
  gitUrl, // the url of the repository
  projectName, // if the project for the repository does not exists, a new project will be created with this name
  organizationId, // organization id
  env: {
    KEY_1: "value1",
  },
  provider: ProviderEnum.GITHUB, // the provider of the git url
  branch: "main", // the branch name that should be deployed
  protocol: ProtocolEnum.IPFS, // the protocol on which the deployment should be uploaded
  configuration: {
    framework: FrameworkEnum.REACT,
    workspace: "",
    installCommand: "yarn install",
    buildCommand: "yarn build",
    publishDir: "build",
    nodeVersion: NodeVersionEnum.V_16,
  },
});
```

Params:

- `gitUrl`: the url of the repository.
  <Callout type="info">
    **NOTE:** Only repositories for which you have authorized the Spheron Git
    App can be deployed.
  </Callout>
- `projectName`: if the project for the `gitUrl` does not exists, a new project will be created and `projectName` will be used for the name of the project.
- `organizationId`: the id of the organization.
- `env`: an object that represent environment variables that should be used when deploying. If new variables are passed, they will automatically be added to the project's environment variables. Besides this variables, the variables of the Project will also be added to the deployment process.
- `provider`: the provider of the repository.
- `branch`: the branch of the deployment.
- `protocol`: the protocol on which the deployment should be uploaded.
- `configuration`: the configuration that should be used for the deployment. Checkout [configuration interface](/sdk/site/#get-1) for more details.

Returns the information about the deployment:

```js
interface DeploymentResponse {
  success: boolean;
  message: string;
  deploymentId: string;
  projectId: string;
  deployment: Deployment;
}
```
