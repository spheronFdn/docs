# Cluster

## Create a cluster

### Import from Docker Hub

Spheron allows you to create a new cluster with a custom docker image.
To create a cluster:

1. Click **New Cluster** on the top right corner.
2. Select **Import from Docker Hub**.
3. Enter the names for your cluster and docker image.
4. Then, Add the tag and Click **Next**.
5. Select the instance plan that suits your needs and Click **Select Plan**.
6. Click **Deploy** to initiate deployment without additional configuration.

<Callout>
  **Troubleshoot:** You can change your plan by clicking Change Plan above the
  selected plan.
</Callout>

<Callout type="info">
  **NOTE:** You can Click Deploy after completing any of the steps under
  Additional Configuration to initiate deployment.
</Callout>

#### Additional Configuration (Optional)

**Port Mapping**

1. Click **New Port Mapping** under `Additional Configuration` to create a new port policy mapping.
2. Add the container port, and Select the exposed port you want to map it to.

**Environment Variable**

1. Click **New Variable** under `Additional Configuration` to add new environment variable.
2. Add key and value. Use the **Secret Key** toggle if the value is a secret key.

**Region**

1. Select the region of your choice in the **Region** section under `Additional Configuration`.

**Advanced Configuration**

1. Click **Advanced Configuration** under `Additional Configuration` if you want add advanced configuration.
2. Click **New Command** to add a new command.
3. Click **New Argument** to add a new argument.

**Health Checkup**

1. Click **Health Checkup** under `Additional Configuration` if you want to configure Health Checkup.
2. Add the desired health check path and health check port.

### Start from Marketplace App

Spheron allows you to deploy a compute instance pre-installed with various applications, all set up and ready to use on the Akash Network.
To deploy a marketplace app:

1. Click **New Cluster** on the top right corner.
2. Select **Select marketplace ap**.
3. Pick your desired template from the marketplace.
4. Select the instance plan that suits your needs and Click **Select Plan**.
5. Click **Deploy** to initiate deployment without additional configuration.

<Callout>
  **Troubleshoot:** You can change your plan by clicking Change Plan above the
  selected plan.
</Callout>

<Callout type="info">
  **NOTE:** You can Click Deploy after completing any of the steps under
  Additional Configuration to initiate deployment.
</Callout>

#### Additional Configuration (Optional)

**Region**

1. Select the region of your choice in the **Region** section under `Additional Configuration`.

**Root Info**

1. Add root username and password under their respective sections.

## Verify Installation

The Marketplace App can be accessed only after the Compute Instance is provisioned.
Thus, you need to wait for the installation to complete before you can start using the app.
Most Marketplace Apps can be verified for successful installation using the instructions below, while others may require different procedures.

- **Attempt to access the app** \
  A Marketplace App has an estimated deployment time of about 1-2 minutes.
  If you can successfully access it, the installation has been completed successfully.
  You can connect to the app with a **reverse DNS link** provided if the app has an HTTP port.
  If not, you can connect using the external port of the app, that will also be provided after the instance is provisioned.
- **Check instance logs and events** \
  After successfully deploying a Marketplace App, it will produce logs and events, which you can check for any issues or errors.
